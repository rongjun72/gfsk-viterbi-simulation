# Trellis 状态转移详解

## 一、为什么状态数是 4？

**信道模型**：3径 ISI，h = [0.8, 0.5, 0.3]

$$r_k = 0.8 \cdot s_k + 0.5 \cdot s_{k-1} + 0.3 \cdot s_{k-2} + n_k$$

接收信号 r_k 取决于 **3 个连续符号**：当前符号 s_k + 前两个符号 s_{k-1} 和 s_{k-2}。

为了计算「期望输出」，我们需要知道 s_{k-1} 和 s_{k-2} 是什么。这两个历史符号的所有可能组合，就是**状态**。

- BPSK 符号只有 2 种：+1（对应比特 0）或 -1（对应比特 1）
- 2 个历史符号，每个 2 种 → 状态数 = 2² = **4**

---

## 二、状态的二进制编码

状态用 **(L-1) = 2** 位二进制编码：

```
状态索引 = (b_{k-1} << 1) | b_{k-2}

b_{k-1}: 前1个符号（s_{k-1}）的比特值
b_{k-2}: 前2个符号（s_{k-2}）的比特值

bit=0 → 符号 +1
bit=1 → 符号 -1
```

| 状态索引 | 二进制 (b_{k-1} b_{k-2}) | 符号序列 [s_{k-1}, s_{k-2}] | 简写 |
|---------|--------------------------|---------------------------|------|
| 0 | 00 | [+1, +1] | ++ |
| 1 | 01 | [+1, -1] | +- |
| 2 | 10 | [-1, +1] | -+ |
| 3 | 11 | [-1, -1] | -- |

> ⚠️ **注意**：代码里状态位的排列是 `b_{k-1}` 在高位（第1位），`b_{k-2}` 在低位（第0位），所以状态2(10)对应的是 s_{k-1}=-1, s_{k-2}=+1。

---

## 三、状态转移规则（核心！）

**时刻 k 的状态**：state = [s_{k-1}, s_{k-2}]

**输入新符号 s_k**（比特 0 或 1）→ 符号 +1 或 -1

**期望输出** = 当前3个符号卷积的结果：

$$\text{expected} = h_0 \cdot s_k + h_1 \cdot s_{k-1} + h_2 \cdot s_{k-2}$$

**下一状态** = [s_k, s_{k-1}]（新符号进来，最旧的符号被挤出去）

---

## 四、逐个验证 8 条转移

### 状态 0：历史 = [+1, +1]（++）

**输入 bit=0 → s_k = +1**
- 期望输出 = 0.8×(+1) + 0.5×(+1) + 0.3×(+1) = 0.8 + 0.5 + 0.3 = **1.600** ✅
- 下一状态 = [+1, +1] → 二进制 00 → **状态 0** ✅

**输入 bit=1 → s_k = -1**
- 期望输出 = 0.8×(-1) + 0.5×(+1) + 0.3×(+1) = -0.8 + 0.5 + 0.3 = **0.000** ✅
- 下一状态 = [-1, +1] → 二进制 10 → **状态 2** ✅

---

### 状态 1：历史 = [+1, -1]（+-）

**输入 bit=0 → s_k = +1**
- 期望输出 = 0.8×(+1) + 0.5×(+1) + 0.3×(-1) = 0.8 + 0.5 - 0.3 = **1.000** ✅
- 下一状态 = [+1, +1] → 二进制 00 → **状态 0** ✅

**输入 bit=1 → s_k = -1**
- 期望输出 = 0.8×(-1) + 0.5×(+1) + 0.3×(-1) = -0.8 + 0.5 - 0.3 = **-0.600** ✅
- 下一状态 = [-1, +1] → 二进制 10 → **状态 2** ✅

---

### 状态 2：历史 = [-1, +1]（-+）

**输入 bit=0 → s_k = +1**
- 期望输出 = 0.8×(+1) + 0.5×(-1) + 0.3×(+1) = 0.8 - 0.5 + 0.3 = **0.600** ✅
- 下一状态 = [+1, -1] → 二进制 01 → **状态 1** ✅

**输入 bit=1 → s_k = -1**
- 期望输出 = 0.8×(-1) + 0.5×(-1) + 0.3×(+1) = -0.8 - 0.5 + 0.3 = **-1.000** ✅
- 下一状态 = [-1, -1] → 二进制 11 → **状态 3** ✅

---

### 状态 3：历史 = [-1, -1]（--）

**输入 bit=0 → s_k = +1**
- 期望输出 = 0.8×(+1) + 0.5×(-1) + 0.3×(-1) = 0.8 - 0.5 - 0.3 = **0.000** ✅
- 下一状态 = [+1, -1] → 二进制 01 → **状态 1** ✅

**输入 bit=1 → s_k = -1**
- 期望输出 = 0.8×(-1) + 0.5×(-1) + 0.3×(-1) = -0.8 - 0.5 - 0.3 = **-1.600** ✅
- 下一状态 = [-1, -1] → 二进制 11 → **状态 3** ✅

## 六、一个具体例子：走一遍 Trellis

**假设发送比特序列**：0, 1, 0, 0 （对应符号 +1, -1, +1, +1）

**从状态 0 开始**（假设前两个符号都是 +1）

### 时刻 0：输入 bit=0（s_0 = +1）
- 当前状态：0 = [+1, +1]（初始假设）
- 期望输出：0.8×(+1) + 0.5×(+1) + 0.3×(+1) = **1.600**
- 下一状态：0 = [+1, +1]（新输入 +1，旧历史 +1）

### 时刻 1：输入 bit=1（s_1 = -1）
- 当前状态：0 = [+1, +1]（上一时刻的下一状态）
- 期望输出：0.8×(-1) + 0.5×(+1) + 0.3×(+1) = **0.000**
- 下一状态：2 = [-1, +1]（新输入 -1，旧历史 +1）

### 时刻 2：输入 bit=0（s_2 = +1）
- 当前状态：2 = [-1, +1]
- 期望输出：0.8×(+1) + 0.5×(-1) + 0.3×(+1) = **0.600**
- 下一状态：1 = [+1, -1]（新输入 +1，旧历史 -1）

### 时刻 3：输入 bit=0（s_3 = +1）
- 当前状态：1 = [+1, -1]
- 期望输出：0.8×(+1) + 0.5×(+1) + 0.3×(-1) = **1.000**
- 下一状态：0 = [+1, +1]（新输入 +1，旧历史 +1）

**完整路径**：0 → 0 → 2 → 1 → 0，对应输入比特：0, 1, 0, 0

## 七、Viterbi 怎么利用这个来检测？

实际接收时，我们不知道发送的是哪个序列。Viterbi 的做法是：

1. **对每个时刻**，每个状态都保留一条「幸存者路径」——到该状态累计距离最小的路径
2. **分支度量** = |r_k - expected|²，即接收信号与期望输出的距离
3. **累计度量** = 之前所有时刻的分支度量之和
4. 最终选择累计度量最小的状态，**回溯**幸存者路径，得到检测的比特序列

这就是动态规划：不需要穷举所有 $2^N$ 个序列，只需在每个时刻每个状态保留一条最优路径，复杂度从 $O(2^N)$ 降到 $O(N \cdot 2^L)$。

---

*总结一句话：状态就是「最近 L-1 个符号的历史」，转移就是「新符号进来、最旧符号出去、期望输出等于卷积结果」。*
